Plug into Claude (MCP) Pro+ What's MCP? ↗

MCP (Model Context Protocol) is how Claude Desktop talks to outside services. Drop these snippets in and Claude can list, edit, and publish your pages from inside any conversation. The official tadaify MCP server (@tadaify/mcp) is open-source and runs locally.

One-liner — add tadaify MCP to Claude Code
$ claude mcp add tadaify --url https://api.tadaify.com/mcp \ --header "Authorization: Bearer YOUR_KEY"
npm install for self-hosted server (optional)
$ npm install -g @tadaify/mcp
Claude Desktop config — ~/Library/Application Support/Claude/claude_desktop_config.json
{ "mcpServers": { "tadaify": { "command": "tadaify-mcp", "env": { "TADAIFY_API_KEY": "sk_tdf_•••…x9k2" } } } }
⚡
The MCP package @tadaify/mcp ships separately at github.com/graspsoftwarepw/tadaify-mcp. The config above is ready for when it lands — copy buttons still work today.
Docs & examples: docs.tadaify.com/api/mcp ↗
Set up Claude Desktop in 4 steps
  1. Open Claude Desktop → Settings → Developer → Edit config. If "Developer" isn't visible, enable it under Settings → General → Show developer options.
  2. Paste the JSON above. Replace sk_tdf_•••…x9k2 with your real key — generate one above if you haven't yet.
  3. Save and quit Claude Desktop. Re-open it — the tadaify tool icon (🔑) should appear in the conversation footer.
  4. Try a prompt: "What's on my main page right now?" — Claude reads your blocks live. Then try "Add a 'New album out!' announcement at the top."
Custom GPT template (ChatGPT) Pro+

Build your own ChatGPT Custom GPT that reads from and writes to your tadaify page in one click. The JSON below is the OpenAPI 3.1 schema ChatGPT's "Configure → Actions" expects, with auth pre-filled for our REST API. Pair it with your generated key and the MCP config above covers Claude.

OpenAPI 3.1 schema — paste into Configure → Actions → Schema
openapi: 3.1.0 info: title: tadaify Creator API version: "1.0" description: Read and update your tadaify pages, blocks, and subscribers. servers: - url: https://api.tadaify.com/v1 paths: /pages: get: operationId: listPages summary: List all pages on this account responses: { "200": { description: OK } } /pages/{slug}/blocks: get: operationId: listBlocks parameters: - { name: slug, in: path, required: true, schema: { type: string } } post: operationId: addBlock requestBody: content: application/json: schema: { $ref: "#/components/schemas/Block" } components: securitySchemes: tadaifyBearer: type: http scheme: bearer security: [{ tadaifyBearer: [] }]
Authentication settings (paste below the schema)
# In ChatGPT Custom GPT builder → Configure → Authentication: Authentication Type: API Key Auth Type: Bearer API Key: sk_tdf_<paste your key>
Make your Custom GPT in 3 steps
  1. Go to chatgpt.com/gpts/editor and start a new GPT. Requires ChatGPT Plus or Team. Name it something like "My tadaify page assistant".
  2. Open Configure → Actions → Create new action. Paste the schema above and the auth block below it. Set Authentication Type to API Key and your key as the bearer token.
  3. Save and try it. Ask: "List my pages." The GPT should call listPages and respond with your slugs. Want examples that actually publish? Browse docs.tadaify.com/api/gpt-recipes ↗
