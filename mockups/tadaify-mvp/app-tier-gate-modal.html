✨ TierGate modal — shared upsell flow at save

The single shared upsell modal that appears when a creator clicks Save on a feature that needs a higher tier. Pick a trigger below to open the modal with the matching content + state. Premium features stay fully visible + interactive in the editor — the gate fires only here, at save.

1Trigger by feature

2Force a state inside the open modal

Open a trigger first, then use the state strip at the top of the modal to flip between states for that trigger.

How it works in production. A creator on Free configures A/B testing on a Newsletter block — fully visible, fully interactive (per feedback_no_blur_premium_features). They click Save. The block-editor calls TierGate.checkAndProceed({features, currentTier, …}). If currentTier >= max(features.requires) → save proceeds silently (toast). Otherwise this modal opens with the trigger context + the lowest-tier-that-covers-everything panel + the reassurance + (optional) save-without partial branch.

Yearly toggle. Annual saves 2 months (10/12 of monthly). Defaults to whatever pricing.html is showing — annual in the canonical mockup.

This is the ONLY upsell scene in tadaify. No sticky banner (per AP-031), no "you're missing out" emails, no popups after N days. Premium features are visible everywhere. The gate is one friendly modal at one moment.
