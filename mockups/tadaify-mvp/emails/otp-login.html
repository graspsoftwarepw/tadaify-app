Spec notes — what differs from signup OTP

  • Greeting: Welcome back, @<handle> — handle is always known on the login path (DEC-307), so no fallback copy needed.
  • Sub-line: signing you in to tadaify instead of welcome to tadaify.
  • Subject: Sign in to tadaify: {{ .Token }} — set via config.toml [auth.email.template.magic_link] (Supabase reuses the magic-link slot for OTP-only login).
  • Device card: shows browser + OS + city + UTC time; helps spot rogue sign-in attempts. Same data Supabase exposes via the auth event payload — values mocked in this preview.
  • No magic link: still OTP-only — the device card is informational, not clickable.
  • Token rendering, dark-mode parity, plain-text fallback, trust strip: identical to signup OTP — see otp-signup.html.
