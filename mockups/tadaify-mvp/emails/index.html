Auth emails — Phase 1

Two transactional emails cover 100% of the Phase 1 auth surface: the OTP sent on signup (new visitor confirms their address) and the OTP sent on every subsequent login (passwordless sign-in is the only login path in tadaify). Token-only delivery per TR-tadaify-002 — the email body never contains {{ .ConfirmationURL }} or any clickable auth link. The 6-digit code is the entire auth handshake. Each mockup shows the email rendered side-by-side in a light client and a dark client, plus the text/plain multipart fallback below.
