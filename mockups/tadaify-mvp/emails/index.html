Older transactional email gallery (10 templates: signup verify, magic link, reset, invite, payment-failed, etc.) lives at legacy-gallery.html — predates the Slice B OTP-only contract and is being superseded as each Phase 1/2 mockup lands.

Slice B contract these mockups demonstrate

  • OTP-only: exactly one CTA per email — the 6-digit code. No magic link, no {{ .ConfirmationURL }} button (DEC-294).
  • Multipart MIME: every template ships text/html + text/plain alternative parts (TR-tadaify-002).
  • No external assets: wordmark is styled text; trust strip is unicode emoji + text. Email body fetches no font / no image CDN (ECN-150-02).
  • Dark-mode parity: Apple Mail dark / Gmail dark render kept readable via the dark variant block in each template (ECN-150-03).
  • Token disambiguation: Crimson Pro display + font-feature-settings: "tnum","lnum" so 0/O and 1/l/I read cleanly (ECN-150-06).
  • config.toml wiring: signup → [auth.email.template.signup]; login OTP → [auth.email.template.magic_link] (Supabase reuses that slot for OTP-only login).
  • Trust strip: 🔒 Price locked for life · 💸 30-day money-back · 🛡 GDPR-compliant — same chips that appear under the register form on register.html.
