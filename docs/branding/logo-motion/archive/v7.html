Implementation note

Single @keyframes orbSway on one <circle> element. animation-timing-function: ease-in-out is set on the animation shorthand — this applies to each keyframe-to-keyframe segment individually, so every glide (left, L→R sweep, return) decelerates smoothly at both ends. No JS. No spring. No overshoot. transform-origin: 50px 50px (SVG viewBox coords, not percent) keeps the orb anchored to the large orb's centre plane.

Why v7 should work

  • No more "za wolno". At 5.5s, any 3-second glance catches at least one full sway — the orb is visibly moving no matter when you look.
  • No more "galareta". Stillness is only 0.8s per cycle (15%) vs. v6's 10+ seconds (70%+). The orb reads as alive from moment one.
  • Matches user's exact request. Pattern is literally left → right → center (lewo → prawo → środek) with brief beats, not pendulum ping-pong and not a step-and-stare.
  • Still premium. 0.2s beats at extremes (not instant reversal) preserve the "present" feel. ease-in-out on all segments — no bounce, spring, or overshoot.
  • No new axes. Pure translateX only. No rotation, scale, opacity, or colour change.

Honest tradeoffs

  • Tempo vs. premium tension. 5.5s is noticeably more active than v6. On pages with competing fast motion (video autoplay, animated illustrations) it may feel less premium. On clean product pages it should read as "alive and confident".
  • 0.8s total rest per cycle. Just enough for the logo-at-rest moment to register; not long enough to feel frozen. If user wants more stillness, 700ms rest (vs 400ms) would push the ratio without tipping back toward "galareta".
