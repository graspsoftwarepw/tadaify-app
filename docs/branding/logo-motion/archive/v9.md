## The fix

- **Motion kept intact.** All v8 JS orbit engine parameters are preserved verbatim: 360¬į clockwise orbit, 8.2s cycle, radial breathing ¬Ī0.55 SVG units at 4 cycles/orbit, 0.9s ease-out glide in, 0.9s ease-in return, 0.8s center rest. Not a single timing constant changed.
- **Orb style restored from v1 mockup.** `tadaify-logo-motion-mockup.html` (the original v1) and `brand-exploration.html` used layered gradient + inner rim highlight + glow drop-shadow on both orbs. v8 had regressed to flat single-color fills (`#6366F1` purple, `#F59E0B` warm). v9 ports those styles into SVG `linearGradient`/`radialGradient` definitions that work with the SVG circle approach the orbit engine requires.

### React/SVG implementation note

In React, each orb is two `<circle>` elements sharing the same animated `cx`/`cy`:
```tsx
// Purple stage orb (static, no position animation)
<circle cx={50} cy={50} r={40} fill="url(#stageLinear)" />
<circle cx={50} cy={50} r={40} fill="url(#stageHighlight)" />
<circle cx={50} cy={50} r={40} fill="none"
        stroke="rgba(255,255,255,0.28)" strokeWidth={3.5} />

// Warm creator orb (animated ‚ÄĒ both circles share same cx/cy ref)
<circle cx={orbX} cy={orbY} r={20} fill="url(#warmLinear)" />
<circle cx={orbX} cy={orbY} r={20} fill="url(#warmHighlight)" />
```

All gradient defs live in a single `<defs>` block. Filter `id`s must be unique per SVG instance ‚ÄĒ scope with a prop-injected prefix if rendering multiple logo sizes on the same page.

## A/B question back to user

**Drop shadow intensity ‚ÄĒ should the glow be subtle (0.15‚Äď0.25 opacity, current default, closest to v1 mockup) or slightly richer (0.35‚Äď0.42 opacity)?**

- **Subtle (current):** `rgba(79,70,229, 0.25)` purple / `rgba(245,158,11, 0.28)` warm. Dimensional without feeling heavy. Works on light and dark backgrounds equally. This is what the v1 mockup used.
- **Richer:** `rgba(79,70,229, 0.40)` purple / `rgba(245,158,11, 0.42)` warm. More "lit stage spotlight" energy ‚ÄĒ the orbs feel genuinely glowing. Better in dark contexts; can feel slightly heavy on the white page background.

Default shipped in v9 is **subtle** (matching v1 mockup). If you want richer, it's a two-line change on the `feDropShadow flood-color` values.
