# tadaify — Logo Motion v5: Subtle Sway

## Metaphor retained

Yellow orb = creator. Purple orb = stage.

In v4 the creator was *greeting* — walking to each corner, waving to the audience. That reading is correct but too busy for a logo mark. In v5 the creator is simply *present on stage*, swaying gently. Less greeting, more existing. The energy is: quiet confidence, not showmanship.

---

## Why v4 failed

v4's "wave-at-edges" motion had four distinct phases (N → E → S → W), each with its own travel arc and a perpendicular rocking wave. The combined effect:

- **Too many events per cycle.** Four visits × (travel + wave + return) = 12 sub-movements in 11.6 s. The eye is never allowed to rest.
- **Amplitude too large.** The warm orb traveled all the way to the edge of the large orb (±20 SVG units = 100% of the available travel radius), making it feel like the orb was pushing to escape.
- **Waving itself is attention-grabbing.** The perpendicular rocking at each edge was the primary visual event — it read as "look at me" rather than "I am here."

User feedback verbatim: *"Zbyt chaotyczny ten ruch machający. To ma być takie subtelne, delikatne, powolne."* (Too chaotic, the waving motion. It should be subtle, delicate, slow.)

---

## Why v5 is clean

One axis. One glide. One rest. That is the entire motion vocabulary.

- **Single horizontal axis only.** `translateX` only. No Y, no rotation, no scale.
- **Amplitude of ±8 SVG units** — 20% of the large orb radius. The warm orb stays well inside the purple boundary at all times. At peak displacement the warm orb edge is 22 px from the canvas left edge; the large orb edge is at 10 px. There is 12 px of visible purple border between them.
- **Long rest.** 25% of the 9 s cycle (2.25 s) is complete stillness at center. The logo reads as static unless the viewer watches it for several seconds.
- **Pure CSS.** Single `@keyframes orbSway` block + one `animation` declaration. No JavaScript, no WAAPI. Deterministic, zero overhead.

## Full timing table

| Keyframe | Time (9 s cycle) | Position | Description | Easing |
|---|---|---|---|---|
| 0% | 0 s | `translateX(0)` | Center — cycle start (also post-rest) | ease-in-out |
| 20% | 1.8 s | `translateX(−8px)` | Full left — warm orb 42 px from canvas center | ease-in-out |
| 55% | 4.95 s | `translateX(+8px)` | Full right — slow cross-stage glide (~3.1 s) | ease-in-out |
| 75% | 6.75 s | `translateX(0)` | Center — orb settles back | ease-in-out |
| 100% | 9.0 s | `translateX(0)` | Center hold / rest (2.25 s stillness) | — |

Total cycle: **9.0 s**. Motion phase: 6.75 s. Rest phase: 2.25 s.

---

## Accessibility

`prefers-reduced-motion: reduce` → `animation: none`. Warm orb remains stationary at center. The logo is indistinguishable from the static brand mark. No JavaScript required to implement this — the CSS media query handles it natively.

The wordmark `tada!ify` **never** animates in any variant or size.

---

## Implementation hint

### CSS (production-ready as-is)

```css
@keyframes orbSway {
  0%   { transform: translateX(0);    }
  20%  { transform: translateX(-8px); }
  55%  { transform: translateX(8px);  }
  75%  { transform: translateX(0);    }
  100% { transform: translateX(0);    }
}

.orb-warm {
  animation: orbSway 9s ease-in-out infinite;
  transform-origin: 50px 50px; /* center of the SVG viewBox */
}

@media (prefers-reduced-motion: reduce) {
  .orb-warm { animation: none; }
}
```

Note: `transform-origin` must reference the center of the **large orb** in SVG user units (50px 50px for a `viewBox="0 0 100 100"` layout), not the geometric center of the warm orb circle itself.

### Framer Motion (React)

```tsx
<motion.circle
  cx={50} cy={50} r={20}
  animate={{ x: [0, -8, 8, 0, 0] }}
  transition={{
    duration: 9,
    times: [0, 0.20, 0.55, 0.75, 1.0],
    ease: 'easeInOut',
    repeat: Infinity,
    repeatType: 'loop',
  }}
/>
```
