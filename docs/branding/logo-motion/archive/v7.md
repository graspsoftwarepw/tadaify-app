# Logo motion v7 — continuous gentle sway

## User feedback history

| Version | Model | User verdict |
|---------|-------|-------------|
| v4 | — | "za chaotyczne, za dużo machania" (too chaotic, too much waving) |
| v5 | Sonnet | "za wolno" (too slow) — 9s cycle, 2.25s rest |
| v6 | Opus | "galareta" (jelly) — 14s cycle, 9.7s long rest, 70% stillness |

Both v5 and v6 failed in opposite directions. v5 moved too slowly (9s cycle, viewer could miss entire sweeps). v6 barely moved at all (70% of the time the orb was static — "jelly that doesn't move").

## The fix

**Continuous sway. 5.5s cycle. Short rests.**

Pattern per user's verbatim request: "lewo, prawo, powrót do środka" (left, right, back to center).

The warm orb glides left, pauses briefly (0.2s — a "beat", not a hold), sweeps through to the right, pauses briefly (0.2s), then returns to center with a short rest (0.4s) before looping. At 5.5s total, any 3-second casual glance catches at least one full sway.

## Full timing table

| Phase | Time range | Duration | Displacement | Easing |
|-------|-----------|----------|-------------|--------|
| Rest at center | 0.0 → 0.4 s | 0.4 s | 0 | — |
| Glide center → left | 0.4 → 1.7 s | 1.3 s | 0 → −8 px | ease-in-out |
| Hold at left | 1.7 → 1.9 s | 0.2 s | −8 px | — |
| Glide left → right | 1.9 → 3.6 s | 1.7 s | −8 → +8 px | ease-in-out (through center, no stop) |
| Hold at right | 3.6 → 3.8 s | 0.2 s | +8 px | — |
| Glide right → center | 3.8 → 5.1 s | 1.3 s | +8 → 0 px | ease-in-out |
| Rest at center (tail) | 5.1 → 5.5 s | 0.4 s | 0 | — |
| **Total cycle** | — | **5.5 s** | **±8 px = 20% of r=40** | `animation: orbSway 5.5s ease-in-out infinite` |

Motion occupies ~4.7s of a 5.5s cycle = **85% active**. v6 was 31% active. v5 was 75% active but at half the frequency.

## Why this should be acceptable

- **No more "za wolno"**: cycle is 5.5s (vs. 9s). Viewer sees full motion within 3s of any glance.
- **No more "galareta"**: total center-rest is 0.8s per cycle (15%) vs. v6's 9.7s (70%). The orb is always going somewhere.
- **Matches verbatim request**: left → right → center, with short beats at extremes.
- **Still premium**: 0.2s beats (not instant reversals), ease-in-out on all segments, no bounce, no spring, no overshoot, no rotation, no scale change.

## What changed vs v5 and v6

| Dimension | v5 | v6 | v7 |
|-----------|----|----|-----|
| Cycle | 9.0 s | 14.0 s | **5.5 s** |
| Center rest | 2.25 s | 9.7 s | **0.4 s** |
| Hold at extremes | none | 2.2 s | **0.2 s** |
| Motion ratio | 75% | 31% | **85%** |
| Pattern | pendulum ±sway | one-direction step/cycle | **L→R→C per cycle** |
| Warm orb count | 1 | 2 (stacked, offset phase) | **1** |

## Implementation

Pure CSS. No JS. No external dependencies beyond Google Fonts.

```css
@keyframes orbSway {
  0%    { transform: translateX(0px);  }   /* 0.0s — rest */
  7.3%  { transform: translateX(0px);  }   /* 0.4s — rest ends */
  30.9% { transform: translateX(-8px); }   /* 1.7s — full left */
  34.5% { transform: translateX(-8px); }   /* 1.9s — left beat ends */
  65.5% { transform: translateX(8px);  }   /* 3.6s — full right */
  69.1% { transform: translateX(8px);  }   /* 3.8s — right beat ends */
  92.7% { transform: translateX(0px);  }   /* 5.1s — back at center */
  100%  { transform: translateX(0px);  }   /* 5.5s — loop */
}

.orb-warm {
  animation: orbSway 5.5s ease-in-out infinite;
  transform-origin: 50px 50px;
}
```

`animation-timing-function: ease-in-out` applies per keyframe segment — each glide decelerates smoothly at both ends. The left→right sweep (1.7s, −8→+8px) passes through center organically without stopping — the acceleration/deceleration curve creates a natural "sway" feel rather than ping-pong.

`transform-origin: 50px 50px` uses SVG viewBox coordinates (not percentages) so the orb stays anchored to the large orb's center regardless of the rendered pixel size.

`prefers-reduced-motion: reduce` collapses to static centered mark.

Favicons (32px, 16px) are always static — `animation: none !important` via `.static` class.

## Geometry

- SVG viewBox: `0 0 100 100`
- Large purple orb: `cx=50 cy=50 r=40`
- Warm amber orb: `cx=50 cy=50 r=20` (= large orb RADIUS, per brand-lock.md §4)
- Amplitude: ±8 SVG units = 20% of r=40
- At full left: warm orb left edge = 42−20 = 22 (large orb left = 10). Warm orb stays inside purple.
- At full right: warm orb right edge = 58+20 = 78 (large orb right = 90). Warm orb stays inside purple.
