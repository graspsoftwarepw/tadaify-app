# Logo Motion v2 — Center-Start Greeting Animation

## Core Principle: Always Start at Center

The warm orb's home position is **center** — overlapping the large purple orb.
This is the locked static brand mark (DEC-007). Every loop of the animation
must depart from that position and return to it, so:

- The static logo and the animated logo are **the same mark** at rest.
- A user who sees the favicon, then the hero, recognizes the same shape.
- The animation reads as a gesture, not as a loading state.

The warm orb leaves, greets, and comes back. It doesn't orbit, it doesn't spin.
It waves.

---

## Greeting Points

Four audience sections are acknowledged in sequence, one per loop:

| Point | Clock Position | Symbolic Audience Section | Timing |
|-------|---------------|--------------------------|--------|
| GP1 | 11 o'clock (top-left) | Left balcony | 16% → 22% |
| GP2 | 1 o'clock (top-right) | Right balcony | 29% → 35% |
| GP3 | 3 o'clock (stage-right) | Side stalls | 42% → 47% |
| GP4 | 6 o'clock (front-center) | Center front row | 56% → 62% |

**Why this path (not a full orbit):**
- A full 360° orbit feels mechanical — like a loading spinner.
- Visiting 4 distinct points with deliberate pauses reads as intentional greeting.
- The sequence moves left → right → side → front, which mimics a real performer's
  acknowledgement sweep: sides first, then the house, then the closest row.
- GP4 (front/bottom) is the climactic pause — the performer leans toward the
  audience before returning to center.

**Why not 5 or more points:**
- Beyond 4 stops the animation feels rushed at 4.2s total, or too long at 6s+.
- Three points (left, right, front) also works but feels incomplete — the side
  stalls deserve a moment.

---

## Timing Specification

```
Total cycle:  4.2 seconds
Loop:         infinite
Easing:       cubic-bezier(0.45, 0, 0.55, 1.0) for travel legs
              (smooth ease-in-out — not snappy, not sluggish)
```

| Phase | Keyframes | Abs. Duration | Description |
|-------|-----------|---------------|-------------|
| Rest at center | 0% → 10% | 420ms | Static brand position |
| Travel → GP1 | 10% → 16% | 252ms | Move to top-left |
| Pause at GP1 | 16% → 22% | 252ms | Left balcony greeting |
| Travel → GP2 | 22% → 29% | 294ms | Sweep across top to top-right |
| Pause at GP2 | 29% → 35% | 252ms | Right balcony greeting |
| Travel → GP3 | 35% → 42% | 294ms | Drop to stage-right |
| Pause at GP3 | 42% → 47% | 210ms | Side stalls (shorter — less prominent) |
| Travel → GP4 | 47% → 56% | 378ms | Long arc to front-center |
| Pause at GP4 | 56% → 62% | 252ms | Front row — the climactic moment |
| Return arc | 62% → 72% | 420ms | Gentle sweep back through center-left |
| Land at center | 72% → 78% | 252ms | Arrives home cleanly |
| Long rest | 78% → 100% | 924ms | ~1s pause before next loop |

**GP3 pause is intentionally shorter** (210ms vs. 252ms for GP1/GP2/GP4) because
the side stalls get a nod, not a bow. The front-row pause (GP4) is the
emotional peak.

**The return arc (62% → 72%)** passes briefly through a center-left midpoint
rather than going straight back up. This avoids a mechanical reverse-path feel
and makes the return read as a natural "walking back to center stage" motion.

## CSS Implementation Notes

The animation is a single `@keyframes greet-crowd` applied to the warm orb
`<circle>` via the class `.orb-warm-animated`.

```css
.orb-warm-animated {
  transform-origin: 50% 50%;
  transform-box: fill-box;
  animation: greet-crowd 4.2s cubic-bezier(0.45, 0, 0.55, 1.0) infinite;
}
```

Key details:
- `transform-box: fill-box` ensures the transform-origin is relative to the
  circle's own bounding box, not the SVG viewport. Without this, `translate`
  percentages would be relative to the SVG viewport width/height.
- The translate percentages in `@keyframes` are therefore **percentages of the
  orb's own bounding box width/height**, not the SVG viewBox.
- All three size instances (hero 240px, nav 48px, favicon 32px) share the same
  `@keyframes` and scale correctly because the SVG uses a fixed `viewBox="0 0 100 100"`.

## Accessibility

### `prefers-reduced-motion`

```css
@media (prefers-reduced-motion: reduce) {
  .orb-warm-animated {
    animation: none;
  }
}
```

When reduced-motion is preferred, the warm orb stays at center permanently —
the locked static brand mark. The logo remains fully recognizable and branded;
no information is lost by removing the motion.

No alternative animation (slower, simpler) is offered — the static mark
IS the correct reduced-motion state because it matches the favicon, the
OG image, and every print/static surface.

### ARIA

The SVG elements carry `role="img"` and `aria-label="tada!ify logo"`.
The motion is purely decorative; no `aria-live` or motion-relevant ARIA
attributes are needed.

### Focus states

The logo mark itself is not interactive (not a link or button). If wrapped
in a link (e.g., nav logo → home), the parent `<a>` receives focus styles;
the SVG inside is `aria-hidden="true"` in that context.

## Favicon Guidance

Motion at 16px and 32px is **disabled by design** (the favicon instances
in the HTML use a plain `<circle>` without the `.orb-warm-animated` class).

Reasons:
1. At 16px, a 12-unit orb moving by 28–36 units across a 100-unit viewBox
   covers only ~4–6 pixels of travel — visually imperceptible.
2. Browser favicon rendering is throttled; CSS animations on `<svg>` favicons
   are unreliable across browsers and OS tab bars.
3. The static centered mark is strong enough at favicon size — two overlapping
   circles, purple + warm, reads clearly.

**Favicon export spec:** export two static PNGs (16×16, 32×32) from the HTML
at t=0% (center position). An `.ico` bundle with both sizes is sufficient for
production; a 180×180 apple-touch-icon from the 240px hero variant (also static)
rounds out the set.

---

## Scale Rationale

| Size | Context | Motion |
|------|---------|--------|
| 240px | Landing hero, About page, brand reel | Full animation |
| 48px | Nav bar, app header | Full animation (subtle at this size) |
| 32px | Favicon, browser tab | Static only |
| 16px | Favicon (compressed) | Static only |

At 48px the travel distances are approximately 13–17px of physical movement.
This is subtle but noticeable — enough to feel alive in a nav bar without
being distracting during reading.
