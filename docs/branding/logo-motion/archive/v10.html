What changed vs v9 delta

v10 additions (non-motion)

  • 4 edge flash spotlights: positioned OUTSIDE the purple stage at N / E / S / W. Each is a radial gradient painted BEHIND the stage in SVG z-order. Fades from white-warm core (rgba 255,255,255,1 at 0% → rgba 245,158,11,0 at 100%) over ~45×38 SVG units per ellipse.
  • Flash sync with orbit: each flash's opacity is driven by angular distance between the warm orb's current angle and the flash's cardinal angle. Formula: opacity = max(0, 1 - (|Δangle| / window)^1.6) where window = 0.55 rad (≈31°). Peak opacity reached at exact cardinal crossing; decays cubically as orb moves away.
  • Motion engine unchanged: same 360° clockwise orbit, same radial breathing, same glide durations, same rest. Flashes ride the existing orbit parameters — zero motion drift.
  • Dark surface adaptation: flash color on dark backgrounds removes the white core (pure amber-gold only) so it reads as hot stage light rather than white burn.
  • Mix-blend-mode: screen on flashes so they brighten the stage rather than occluding — reads as light, not a painted overlay.

Tuning knobs

Slider at the top of the hero lets you dial flash intensity from 0% (off) to 200% (heavy). Default 100%. Try 60% for subtle "blink" vs 140% for "camera flash" energy.

Other tunables in JS (not exposed to UI): flash window width (0.55 rad), decay power (1.6), flash ellipse size (rx=45 ry=38 for horizontal pairs; rx=38 ry=45 for vertical pairs), color stops in radialGradient defs.

Accessibility

  • @media (prefers-reduced-motion: reduce) — orb stays centered, flashes permanently off.
  • Flashes are decorative; always paired with static wordmark.
  • Flash intensity never exceeds opacity 1.0 and is confined to a small area — no full-screen-flash seizure risk.
