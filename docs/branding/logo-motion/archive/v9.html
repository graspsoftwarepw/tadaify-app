What changed vs v8 delta

v9 changes (orb rendering only)

  • Orb visual style: gradient fills + inner rim highlights + soft glow drop-shadows, ported from brand-exploration.html / tadaify-logo-motion-mockup.html (v1 mockup). Purple orb: linear-gradient(145deg, #7C78FF, #5B56E8 58%, #4F46E5) + top-left white radial highlight + inner-rim white stroke + rgba(79,70,229,0.25) drop-shadow. Warm orb: linear-gradient(145deg, #FFD36A, #F59E0B 58%, #D97706) + top-left white radial highlight + rgba(245,158,11,0.28) warm drop-shadow.
  • Motion: UNCHANGED from v8 — same 360° clockwise orbit, same radial breathing (±0.55 SVG units, 4 cycles/orbit), same glide durations (0.9s in / 0.9s out), same rest (0.8s), same total cycle (8.2s). JS engine copied verbatim — zero drift.

A/B question for review

Drop-shadow intensity

Current v9 uses the same glow opacity as the original v1 mockup (rgba(79,70,229, 0.25) purple, rgba(245,158,11, 0.28) warm). These feel present but not heavy — closer to the "polished sphere" look than "glowing element".

Option A — subtle (default, current): Purple glow 0.25 opacity, warm glow 0.28 opacity. Reads as dimensional; well-suited for logo use on any background.

Option B — richer: Purple glow 0.40 opacity, warm glow 0.42 opacity. More "stage spotlight" energy — both orbs feel more lit. Risk: can feel heavy on light backgrounds.

Accessibility

Reduced-motion support

  • @media (prefers-reduced-motion: reduce) — all animation paused, warm orb stays centred (inherited from v8).
  • Favicon sizes (32 px, 16 px) are always static.
  • SVG has aria-label on animated logos; decorative ones use aria-hidden="true".
