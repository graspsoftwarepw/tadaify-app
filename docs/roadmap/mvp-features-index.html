§30. Infrastructure Architecture (locked 2026-04-24)
§30. Infrastructure Architecture (DEC-DOMAIN-01 / DEC-DNS-01 / DEC-FRAMEWORK-01 / DEC-INFRA-MINIMAL-01) locked 2026-04-24▸
  • URL routing (DEC-DOMAIN-01): everything under tadaify.com/ — tadaify.com/<handle> creator hub · tadaify.com/<handle>/p/<slug> product · tadaify.com/app dashboard · tadaify.com/admin admin · tadaify.com/try guest editor (REMOVED DEC-355=C) ·tadaify.com/pricing · tadaify.com/trust Trust Center · tadaify.com/t/<name> template preview · tadaify.com/vs/<competitor> · tadaify.com/for/<niche>
  • Subdomains (separate concerns only): preview.tadaify.com (preview generator, no handle collision) · developers.tadaify.com (API docs) · status.tadaify.com (uptime, 3rd-party) · mail.tadaify.com (Resend SPF/DKIM)
  • DNS (DEC-DNS-01): Cloudflare authoritative — OVH registrar → Cloudflare nameservers. Route 53 NOT used. Cloudflare for SaaS handles creator custom domains separately.
  • Framework (DEC-FRAMEWORK-01): Remix (React Router 7) on Cloudflare Workers runtime. File-based routing. SSR for public pages · CSR for dashboard/admin after first load. Deploy: deploy-cloudflare-pages.yml via Wrangler. main → prod · develop → dev.
  • Backend: Supabase (EU Frankfurt) — Postgres + GoTrue + Edge Functions + Realtime. All user data, billing, Stripe Connect, email subscribers.
  • AWS — minimal only (DEC-INFRA-MINIMAL-01): S3 tdfy-analytics-{env} (Parquet cold analytics) · Athena workgroup · Glue catalog · IAM OIDC role for CI/CD. ~100-150 lines Terraform. NO CloudFront · NO Route 53 · NO ACM · NO Lambda by default.
  • External: Stripe (billing + commerce + Connect + Tax) · Resend (mail.tadaify.com) · OpenAI (AI features via backend proxy)
  • Cloudflare services used: Pages (app hosting) · Workers (SSR + API routes) · R2 (media, zero egress) · Analytics Engine (hot analytics 90d) · for SaaS (creator custom domains) · DNS (authoritative) · Bot Management
