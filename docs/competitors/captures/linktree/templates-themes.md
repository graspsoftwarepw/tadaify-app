## Gallery structure (inferred)

Linktree groups templates into niche-oriented collections rather than
purely aesthetic themes. Example niches referenced on the page:
- Long-distance runners
- Musicians
- Podcasters
- Chefs / food creators
- Fitness coaches
- Non-profits / activism
- Small businesses / local retail
- Educators / course creators
- Photographers / visual artists

Each niche template pre-populates:
- A set of relevant link types (e.g. running → "favorite running tracks,
  diet routines, connect fans to social media") [source:
  Linktree templates page summary via search]
- A matching theme / palette / hero visual
- Sample copy that the creator replaces

## Theme capabilities (what's customisable)

From [help article 5434137 + blog
post](https://linktr.ee/blog/linktree-customization-features):

- **Theme base** — preset palette + button style, pick from a gallery
- **Text** — typography changes at theme level; font options expand on
  paid tiers [unclear — whether font-picker is Starter or Pro]
- **Colors** — background, button, text colors on the "Customizable"
  theme base
- **Buttons** — shape (square / rounded / pill), fill / outline,
  hover states [unclear — hover customisation]
- **Header** — profile image, title, bio text
- **Wallpaper** — upload custom background image or pattern; fallback to
  theme default

## Customisation depth ceiling

Linktree does **NOT** support:
- Arbitrary custom CSS [source: Capterra complaints about
  customisation limits; widely referenced as a gap]
- Custom fonts beyond the Linktree font catalogue (no self-hosted /
  @font-face) [unclear]
- Per-link custom CSS (each link button uses the theme's button style)
- Multiple pages / nested navigation (single-page only; Linktree is a
  flat list)
- Responsive breakpoints (mobile-first; desktop is just a centred
  narrow column) [unclear]
- Dark-mode toggle at the creator's discretion (theme dictates; no
  auto-prefers-color-scheme)

## Mobile preview

Theme editor includes a live mobile preview next to the controls. Users
can see the mobile rendering before publishing [source: Capterra quote
"ability to customize your link page and review how it'll be seen by
visitors before publishing"].

## Third-party Linktree-style templates (market context)

The market for "Linktree clone" templates on other platforms is very
active — a signal of the design appetite Linktree itself does not fully
satisfy:

- **GitHub**: [vitor-antoni/linktree-template](https://github.com/vitor-antoni/linktree-template) — FREE HTML+CSS+JS template
- **GitHub**: [vilenarios/linktree-templates](https://github.com/vilenarios/linktree-templates) — permaweb-ready template collection
- **GitHub topic**: [#linktree](https://github.com/topics/linktree?l=html) — active tag
- **Dribbble**: [linktree designs tag](https://dribbble.com/tags/linktree) — designer-community riffs
- **Envato Elements**: [Linktree WordPress themes](https://elements.envato.com/wordpress/linktree)
- **Webflow**: [free Linktree website templates](https://webflow.com/made-in-webflow/linktree?cloneable=true) — community clones
- **Astro**: [linktree-template theme](https://astro.build/themes/details/linktree-template/)

The existence of this ecosystem tells you:
- Designers consider Linktree's aesthetic ceiling low; they build
  richer alternatives on their own stacks
- Developer-tool users (Webflow, Astro, WordPress) self-serve around
  the customisation ceiling — market signal for creator-focused tools
  that ship with truly flexible theming

## SEO surface

`/s/templates` acts as a high-traffic top-of-funnel SEO page because
niche keywords ("Linktree for musicians", "Linktree for podcasters")
are exactly what new-to-the-category users search. Competitors
(Biotree, UniLink, Campsite.bio) are explicitly trying to rank on the
same template SEO terms [source: multiple "alternatives" posts in
search results].
