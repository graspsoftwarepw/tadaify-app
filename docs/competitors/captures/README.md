# Competitor UI Audit — tadaify

This directory stores screenshot-based competitor audit notes for Tadaify.

Workflow:
- Screens are provided in the Codex chat thread.
- Each competitor has its own folder.
- `notes.md` is the source of truth for observed flows, UI patterns, paywalls,
  onboarding friction, and product ideas worth stealing or avoiding.
- Notes are organized by product modules, not just by chronological screens.

Suggested module structure per competitor:
- `01. Onboarding & Auth`
- `02. Dashboard & Navigation`
- `03. Page Editor`
- `04. Public Page / Profile`
- `05. Analytics`
- `06. Monetization / Billing`
- `07. Audience / CRM / Email`
- `08. Growth / Integrations / Embeds`
- `09. Admin / Team / Agency`
- `10. Design System / UX Patterns`
- `11. Paywalls / Plan Gating`
- `12. Open Questions`

Suggested note structure per documented screen inside a module:
- screen id
- flow / route
- evidence
- layout and visible functionality
- how the flow works
- UI / usability assessment for users
- strategic assessment for tadaify
- irritation risk
- whether it is worth implementing
- how tadaify could do it better
- open questions (optional)
